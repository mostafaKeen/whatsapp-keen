Data Integration Flow

Keen Enterprise LLC | WhatsApp & Bitrix24 Synchronization

Overview

This document outlines the seamless movement of data between Gupshup (WhatsApp Provider) and Bitrix24 (CRM), managed by a custom application hosted on Bluehost.

1
Source: WhatsApp Message

A customer sends a message to the business WhatsApp number. Gupshup receives this message via the Meta infrastructure.

↓
2
Bridge: Gupshup Webhook

Gupshup triggers a POST request (Webhook) containing the message, sender's phone, and media links to the Keen Enterprise app hosted on Bluehost.

↓
3
Processing: Keen Enterprise App

The app (webhook.php) validates the data, identifies the sender, and checks if a Lead exists in Bitrix24 using the phone number.

↓
4
Destination: Bitrix24 CRM

The app calls the Bitrix24 REST API to create/update a Lead and delivers the message directly to the Open Channel chat for the agent to reply.

Key Data Mapping

Data Type Gupshup Field Bitrix24 Field
Sender Identity from (Phone) PHONE / LEAD TITLE
Message Content text.body MESSAGE_TEXT (Open Channel)
Attachments image/video/file FILES (Timeline/Chat)
Delivery Status statuses MESSAGE_STATUS (Delivered/Read)

Technical Environment

  • Hosting: Bluehost (Managed by Keen Enterprise LLC)
  • Security: All transfers use HTTPS/SSL encryption.
  • Automation: Automatic Lead creation for new inquiries.
